How to Play Radio Stations on the Bose SoundTouch Speaker

Two API calls from the app — fetch the station list from the proxy server, then send UPnP commands to the speaker.

⚡
Overview

The app does two things — get the list of radio stations from our proxy server, then tell the speaker to play a station via UPnP. The speaker handles the actual audio streaming on its own.

App
→ GET /api/stations →
Proxy Server
→ returns HTTP stream URLs →
App
→ UPnP SOAP →
Bose Speaker
⚠️
Same WiFi Required The app and the Bose speaker must be on the same local WiFi network for the UPnP commands to reach the speaker.
1
Get Station List from Proxy Server

On app launch (or login), fetch the list of radio stations from the proxy server. Each station includes a ready-to-use HTTP stream URL.

HTTP Request
GET  http://ourserver.com/api/stations
Accept: application/json
Response — 200 OK
[
  {
    "id":       "abc123",
    "name":     "DZMM",
    "url":      "https://stream.dzmm.com.ph/live.mp3",   // original HTTPS source
    "proxyUrl": "http://ourserver.com/stream/abc123"       // ← use this for the speaker
  },
  {
    "id":       "def456",
    "name":     "DZBB",
    "url":      "https://stream.dzbb.com/live.mp3",
    "proxyUrl": "http://ourserver.com/stream/def456"
  }
]
FieldDescription
idUnique station ID
nameDisplay name shown in the app
urlOriginal HTTPS source URL — do NOT send this to the speaker
proxyUrlHTTP proxy URL — send this to the speaker via UPnP
💡
Always use proxyUrl — not url The Bose SoundTouch 10 can only play HTTP streams. The proxy server converts the HTTPS source to HTTP. Always use proxyUrl when sending to the speaker.
2
Tell the Speaker What to Play — SetAVTransportURI

Send a UPnP SOAP request to the speaker with the stream URL. This sets the audio source but does not start playback yet.

📡
Speaker UPnP Endpoint http://<speaker-ip>:8091/AVTransport/Control
Discover the speaker IP via mDNS/Bonjour or let the user configure it. Default port is 8091.
HTTP Request — SetAVTransportURI
POST http://<speaker-ip>:8091/AVTransport/Control
Content-Type: text/xml; charset="utf-8"
SOAPAction:   "urn:schemas-upnp-org:service:AVTransport:1#SetAVTransportURI"

<?xml version="1.0" encoding="utf-8"?>
<s:Envelope xmlns:s="http://schemas.xmlsoap.org/soap/envelope/"
             s:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <s:Body>
    <u:SetAVTransportURI xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
      <InstanceID>0</InstanceID>
      <CurrentURI>http://ourserver.com/stream/abc123</CurrentURI>  <!-- proxyUrl here -->
      <CurrentURIMetaData></CurrentURIMetaData>
    </u:SetAVTransportURI>
  </s:Body>
</s:Envelope>
Expected Response — 200 OK
<u:SetAVTransportURIResponse xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
</u:SetAVTransportURIResponse>
3
Start Playback — Play

After SetAVTransportURI succeeds, send the Play command. The speaker will connect to the stream URL and start playing audio.

HTTP Request — Play
POST http://<speaker-ip>:8091/AVTransport/Control
Content-Type: text/xml; charset="utf-8"
SOAPAction:   "urn:schemas-upnp-org:service:AVTransport:1#Play"

<?xml version="1.0" encoding="utf-8"?>
<s:Envelope xmlns:s="http://schemas.xmlsoap.org/soap/envelope/"
             s:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <s:Body>
    <u:Play xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
      <InstanceID>0</InstanceID>
      <Speed>1</Speed>
    </u:Play>
  </s:Body>
</s:Envelope>
Expected Response — 200 OK
<u:PlayResponse xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
</u:PlayResponse>
🔊
Speaker is now playing After Play responds with 200, the speaker streams audio directly from the proxy server. The app can be closed — the speaker continues playing on its own.
4

To stop the speaker, send a Stop command.

Content-Type: text/xml; charset="utf-8"

D
Sample Code — Dart / Flutter

Complete implementation using the Dart http package.

soundtouch_player.dart
import 'package:http/http.dart' as http;
import 'dart:convert';

// 1. Fetch station list from proxy server
Future<List<Map>> fetchStations() async {
  final res = await http.get(
    Uri.parse('http://ourserver.com/api/stations'),
  );
  return List<Map>.from(json.decode(res.body));
}

// 2 & 3. Play a station on the speaker
Future<void> playOnSpeaker(String speakerIp, String proxyUrl) async {
  final endpoint = Uri.parse(
    'http://$speakerIp:8091/AVTransport/Control',
  );

  // Step 2 — SetAVTransportURI
  await http.post(endpoint,
    headers: {
      'Content-Type': 'text/xml; charset="utf-8"',
      'SOAPAction':
        '"urn:schemas-upnp-org:service:AVTransport:1#SetAVTransportURI"',
    },
    body: '''<?xml version="1.0" encoding="utf-8"?>
<s:Envelope xmlns:s="http://schemas.xmlsoap.org/soap/envelope/"
            s:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <s:Body>
    <u:SetAVTransportURI xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
      <InstanceID>0</InstanceID>
      <CurrentURI>$proxyUrl</CurrentURI>
      <CurrentURIMetaData></CurrentURIMetaData>
    </u:SetAVTransportURI>
  </s:Body>
</s:Envelope>''',
  );

  // Step 3 — Play
  await http.post(endpoint,
    headers: {
      'Content-Type': 'text/xml; charset="utf-8"',
      'SOAPAction':
        '"urn:schemas-upnp-org:service:AVTransport:1#Play"',
    },
    body: '''<?xml version="1.0" encoding="utf-8"?>
<s:Envelope xmlns:s="http://schemas.xmlsoap.org/soap/envelope/"
            s:encodingStyle="http://schemas.xmlsoap.org/soap/encoding/">
  <s:Body>
    <u:Play xmlns:u="urn:schemas-upnp-org:service:AVTransport:1">
      <InstanceID>0</InstanceID>
      <Speed>1</Speed>
    </u:Play>
  </s:Body>
</s:Envelope>''',
  );
}

// Example usage
void main() async {
  final stations = await fetchStations();
  final first = stations[0];

  await playOnSpeaker(
    '192.168.1.10',          // speaker IP
    first['proxyUrl'],        // http://ourserver.com/stream/abc123
  );
}
!
Important Notes
📶
Speaker must be on the same WiFi as the phoneThe UPnP commands go directly from the phone to the speaker over the local network. This will not work over mobile data.
🔌
Port 8091 is the UPnP AVTransport portThe Bose SoundTouch 10 also exposes port 8090 (SoundTouch proprietary API) and port 8080 (WebSocket). For UPnP/DLNA playback of custom URLs, always use port 8091.
✅
After Play — app is not neededOnce playback starts, the speaker streams audio directly from the proxy server. The app and the phone can be closed. The speaker keeps playing until stopped.
