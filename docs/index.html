System Flow

How the app fetches stations and how the speaker plays audio

SoundTouch App
Proxy Server
Bose Speaker
Radio Source

User Account
Web UI
Admin Setup — Web UI manages stations on the server
Web UI — Admin Panel
Admin opens the web interface — browser loads the page from the proxy server
GET http://ourserver.com/
Web UI — Add Station
Admin adds a radio station — Web UI sends it to the proxy server
POST http://ourserver.com/api/stations
{ name: "DZMM", url: "https://stream.dzmm.com.ph/live.mp3" }
Proxy Server
Saves the station — returns the HTTP proxy URL
{ proxyUrl: "http://ourserver.com/stream/dzmm" }
Web UI — Sync to Database
Web UI saves the updated station list to the Database for the app to access
save / update → Database
Database
Station list stored — ready for users to fetch on login
Phase 1 — User Login
SoundTouch App
User logs in — sends credentials to Database
login / auth → Database
Database
Validates user account — returns saved station list
user account → valid
station list → [ DZMM, DZBB, RX93... ]
SoundTouch App
App now has the station list — displayed on screen
Phase 2 — App gets updated proxy URLs from Server
SoundTouch App
Requests updated proxy URLs from the proxy server
GET http://ourserver.com/api/stations
Proxy Server
Returns station list with HTTP proxy URLs
[
 { name: "DZMM", proxyUrl: "http://ourserver.com/stream/dzmm" },
 { name: "DZBB", proxyUrl: "http://ourserver.com/stream/dzbb" }
]
SoundTouch App
App has the proxy URLs — ready to send to speaker
Phase 3 — User taps a station
SoundTouch App — Step 1
Sends proxy URL to speaker via UPnP SOAP
POST http://192.168.1.6:8091/AVTransport/Control
SOAPAction: SetAVTransportURI
CurrentURI: http://ourserver.com/stream/dzmm
SoundTouch App — Step 2
Sends Play command to speaker
POST http://192.168.1.6:8091/AVTransport/Control
SOAPAction: Play
Phase 4 — Speaker connects and plays audio
Bose Speaker
Connects to the proxy server to fetch audio
GET http://ourserver.com/stream/dzmm
Proxy Server
Fetches the HTTPS audio source
GET https://stream.dzmm.com.ph:8000/live.mp3
Radio Source
Returns continuous audio stream
Content-Type: audio/mpeg
→ audio bytes arriving continuously...
Proxy Server
Pipes audio directly to speaker — no download, forwarded immediately
HTTPS source → pipe() → HTTP → speaker
Bose Speaker
Receives audio — now playing
🔊
Speaker plays continuously
App can be closed — speaker talks directly to the proxy server. Phone is no longer needed.
